NOM DU PRODUIT : Compaq Office Server pour OpenVMS V6.0/FRANÇAIS


DESCRIPTION

Compaq(tm) Office Server pour OpenVMS(tm) V6.0 est une amélioration 
des versions précédentes d'Office Server, ALL-IN-1(tm) et MailWorks(tm). 
Il propose des services de messagerie et de collaboration puissants 
et évolutifs à un large ensemble de clients de messagerie sur OpenVMS 
Alpha et VAX, et des possibilités de communication avec un ensemble 
de réseaux de messagerie et d'annuaires. Les principales nouvelles 
fonctions sont les suivantes :

o  Support pour les Classeurs imbriqués, jusqu'à dix niveaux, 
   accessible via les clients Outlook, IMAP4, Web et TeamLinks V5.0.

o  Doublement du nombre de zones partagées de messagerie- SDAF - qui 
   sont maintenant au nombre de dix.

o  Amélioration des performances du serveur IMAP4.

o  Dossiers vides en permanence.

Office Server présente des fonctions de gestion de groupes mixtes 
d'utilisateurs IMAP4, POP3, Windows 2000, Windows ME, Windows NT, 
Windows 95/98, Microsoft[R] Windows[R] V3.11 et V3.1, Apple[R] 
Macintosh[R] et de VT permettant aux petits départements ou à 
l'ensemble de l'entreprise de communiquer, de partager des documents 
et de collaborer au bénéfice d'une plus grande productivité.

Grâce au module de messagerie électronique et au partage des documents 
sur les réseaux grande distance, les utilisateurs peuvent travailler 
ensemble sur des informations multimédia qui ont été créées sur les 
postes de travail, les ordinateurs de départements ou de l'entreprise. 
En outre, grâce à la base serveur et à des fonctions complètes de 
gestion du système, le personnel de gestion nécessaire est limité au 
strict minimum.


SUPPORT CLIENT TeamLinks

Services disponibles pour les clients TeamLinks Windows V3.0, V4.0, 
V4.5 et Macintosh V2.6.2 :

Office Server fournit des services de groupes de travail aux clients 
TeamLinks par l'intermédiaire de liaisons TCP/IP, SLIP, DECnet(tm) 
asynchrones ou AppleTalk[R] (périphériques Macintosh). Le serveur 
permet aux utilisateurs de TeamLinks d'accéder aux tiroirs, classeurs 
imbriqués, documents, messages et listes de distribution des archives 
avec un minimum de formation nécessaire pour les utilisateurs de VT 
déjà familiarisés avec ALL-IN-1 et MailWorks.

Les utilisateurs de postes de travail Windows et Macintosh équipés 
des applications de production de leur choix, disponibles sur ces 
plates-formes, peuvent transférer des documents de tous types 
(y compris des documents multimédia) dans les archives Office Server 
à des fins de stockage, ainsi qu'envoyer ces documents multi-support 
par le service de messagerie, à l'aide des listes de distribution 
existantes.
Le partage des documents intégral permet aux utilisateurs TeamLinks 
d'accéder à tout type de document situé sur les systèmes distants 
Office Server et ALL-IN-1 comme s'ils se trouvaient au niveau local.

Office Server V6.0 permet aux utilisateurs de clients hétérogènes 
(basé sur le Web, IMAP4, POP3, Windows, Macintosh, VT) de coexister 
et de participer pleinement aux activités du groupe de travail.

Pour plus d'informations sur les fonctionnalités offertes par les 
clients TeamLinks, reportez-vous aux Descriptions de Produit Logiciel 
63.25 (Windows) et 48.17 (Macintosh).

Remarque : Le niveau du service client peut être étendu au-delà de 
celui proposé par le client TeamLinks V4.5. Les développeurs 
d'application disposent d'une gamme complète d'interfaces API 
(Application Programming Interfaces) dans le cadre du serveur 
d'archives et du serveur AIDA.


SUPPORT CLIENT MAPI

Office Server fournit des services de messagerie et de travail en 
groupe aux applications utilisant l'interface MAPI (Messaging 
Application Programming Interface), telles que Outlook via le pilote 
MAPI de Compaq Office Server MAPI V6.0 et V7.0, livré séparément. 
La messagerie, les annuaires, les archives, les gestions de groupe, 
le glisser-déplacer et les opérations de conversion de documents sont 
pris en charge sur les systèmes Windows 2000 32-bit, les postes de 
travail Windows NT V4. et les PC sous Windows ME/98/95. Pour plus de 
détails concernant les fonctionnalités proposées, reportez-vous à 
SPD : 61.43.04 pour V7.0 et 61.43.03 pour V6.0.


ACCES AU WORLD WIDE WEB

Les utilisateurs de réseaux Internet ou intranet peuvent répondre, 
ré-acheminer et envoyer des messages électroniques tout en accédant 
à leurs tiroirs, classeurs ou documents personnels - ainsi qu'à tous 
les documents pour lesquels ils disposent d'un droit d'accès - par 
l'intermédiaire d'une connexion et d'un mot de passe via l'interface 
Web Compaq Office Server Web Interface V6.0 et V7.0, livrée séparément.
Les utilisateurs peuvent utiliser des navigateurs tels que Netscape 
Navigator[R] et Microsoft Internet Explorer pour visualiser et 
télécharger sur PC des documents et des messages depuis un café 
Internet, par exemple.

En outre, tout document placé dans un tiroir avec un accès "World Read" 
ou Internet est accessible partout dans le monde sur Internet (sous 
réserve de murs de sécurité et de logiciels d'accès requis). Aucun 
outil spécialisé HTML (Hypertext Markup Language) de modélisation 
d'application n'est requis sur le serveur pour "publier" des documents 
sur Internet. La mise à jour des documents nécessite simplement de 
classer un nouveau document dans les archives pour un accès immédiat 
sur le Web. Pour plus de détails, reportez-vous à SPD : 61,55.04 pour 
V7.0 et 61,55.03 pour V6.0.

Important : La protection de tous les documents contenus dans 
            les archives Office Server peut être assurée par 
            l'intermédiaire des fonctions de sécurité inhérentes. 
            Néanmoins, si ces fonctions ne sont pas utilisées, 
            l'accès à Internet à partir d'Office Server peut 
            générer un risque de violation de sécurité. 
            Compaq décline toute responsabilité concernant la 
            sécurité des documents contenus dans Compaq Office 
            Server.


CLIENT ADMINISTRATOR D'OFFICE SERVER

Le client Administrator peut être installé sur Windows 2000, les 
postes de travail Windows NT V4.0 et les ordinateurs de bureau Intel 
Windows 98/95. Il fournit à l'administrateur système une interface 
utilisateur graphique permettant d'effectuer de nombreuses tâches 
administratives. La possibilité de gérer entièrement le système à partir 
de VT est maintenue, y compris toutes les fonctionnalités de gestion 
héritées d'ALL-IN-1 V3.2.

L'interface utilisateur graphique Administrator permet la gestion 
du système dans les domaines suivants :

o  Gestion des utilisateurs

   - Création de comptes utilisateur

   - Modification de comptes utilisateur

   - Suppression de comptes utilisateur

   - Changement de nom de comptes utilisateur

   - Transfert d'utilisateurs

o  Gestion du serveur

   - Gestion des serveurs d'archives

   - Gestion des serveurs AIDA

   - Configuration des serveurs pour les Pannes et la Répartition des 
     charges

   - Gestion des envois et des appels

o  Gestion de la consignation

o  Procédures de maintenance

o  Gestion générale du système


Le compte client Administrator tente automatiquement de se reconnecter 
au serveur de gestion en cas de redémarrage automatique dû à une panne 
du serveur.

Le client Administrator est commun à l'installation complète d'Office 
Server et à l'option File Cabinet Only (voir ci-dessous).

Remarque : Le compte client Administrator requiert un transport 
           IP et ne prend en charge aucune version de DECnet.


SERVICES DE MESSAGERIE ET D'ANNUAIRES

Office Server est capable de communiquer avec un grand nombre d'agents 
de transfert de messages (MTA) et d'agents de système d'annuaire (DSA).


MAILbus 400

Avec l'intégration des éléments de l'interface X/Open(tm) XAPI et XDS, 
les clients peuvent choisir le système de messagerie MAILbus 400 sur 
système Alpha X.400 (OpenVMS ou Compaq Tru64 UNIX(tm)) et les agents 
de service d'annuaire X.500 Compaq requis comme réseau de messagerie 
et déployer des agents de transfert de messages (MTA) et des agents de 
système d'annuaire (DSA) où et quand ils le souhaitent, avec des 
connexions locales ou éloignées aux systèmes Office Server.


PMDF

Une autre solution consiste à utiliser PMDF de Process Software qui 
fournit un système smtp/MIME fonctionnant sur AlphaServer(tm) et accède 
directement à l'interface MRIF V2.0 d'Office Server, sans qu'il soit 
nécessaire d'utiliser le routeur de messages. PMDF peut utiliser un 
grand nombre de services d'annuaires, y compris son propre (Innosoft) 
Distributed Directory Servier (IDDS) V4.4.1, ainsi que le profil réseau, 
inclus dans Office Server. PMDF ne prend pas en charge les services 
d'annuaires répartis (DDS) du routeur de message.


LDAPv3

Office Server prend en charge le protocole Lightweight Directory 
Access (LDAP) V3, permettant ainsi d'utiliser des services d'annuaires 
LDAPv3 de plus en plus répandus. Sa mise en oeuvre dans Office Server 
V6.0 permet l'utilisation du client VT et des utilitaires de gestion 
inclus dans Office Server pour charger les annuaires LDAP suivants :

o  OpenVMS Enterprise Directory for eBusiness V5.0

o  Tru64 UNIX Enterprise Directory for eBusiness V5.0

o  Process Software (Innosoft) Distributed Directory Server (IDDS) V4.4.1

o  Netscape iPlanet Directory Server V4.0

o  Novell NDS eDirectory 5

La recherche d'adresses est également prise en charge à partir de 
TeamLinks, des clients Outlook, des explorateurs Web, de VT et des 
utilitaires de gestion d'Office Server.


Message Router/MAILbus 1

L'interface MRIF V2.0 étant également incluse, les utilisateurs peuvent 
continuer à utiliser Message Router for OpenVMS. La passerelle 
OpenVMSmail de Message Router permet aux utilisateurs d'Office Server 
d'échanger des messages avec les utilisateurs de l'utilitaire OpenVMS 
Personal Mail. Pour plus de détails, reportez-vous à la Description de 
Produit Logiciel de VAX Message Router for OpenVMS (SPD 26.33).

L'échange de messages avec Internet et les réseaux intranet est possible 
à partir de MAILbus 400, PMDF et de Message Router. Les adresses de type 
Internet sont directement gérées par Office Server.


SERVEUR IMAP4

Le protocole IMAP4rev1 (Internet Messaging Access Protocol 4) est pris 
en charge normalement dans Office Server V6.0. Les utilisateurs peuvent 
accéder à leur classeur ARRIVÉE ou a tout autre dossier à partir de 
clients de messagerie Internet standard, tels qu'Outlook Express et 
Netscape Communicator, à l'aide d'un nom d'utilisateur et d'un mot 
de passe.
Lors de la connexion, les opérations suivantes peuvent être effectuées 
sur les documents et les messages des archives : création, lecture, 
édition, modification des attributs, ré-acheminement, réponse et 
suppression. Le courrier traité est transféré vers un serveur smtp 
"sendmail" prédéfini, tel que Compaq TCP/IP Services ou MultiNet[R] pour 
sa distribution. Office Server V6.0 propose dix niveaux d'imbrication.


SERVEUR POP3

Le protocole Post Office 3 est le protocole standard pris en charge 
par Office Server V6.0. Les utilisateurs accèdent à leur classeur 
ARRIVÉE par l'intermédiaire de clients de messagerie Internet standard, 
tels qu'Eudora et Netscape, en indiquant un nom d'utilisateur et un 
mot de passe. Lors de la connexion, le contenu du classeur Arrivée 
est téléchargé vers le client pour pouvoir être traité hors ligne. 
Lorsque l'utilisateur se reconnecte, le courrier traité est transféré 
vers un serveur "sendmail" prédéfini, tel que Compaq TCP/IP Services 
ou MultiNet[R] pour être distribué. Le serveur POP3 d'Office Server 
offre un grand nombre de fonctions supplémentaires, comprenant :

o  La capacité d'accéder à tout dossier personnel ou à tout tiroir 
   partagé Office Server et de télécharger le contenu sur le client.

o  Gère l'option "delete/retain message on server". L'état des messages 
   restant sur le serveur peut être configuré.

o  Conversion en format texte de messages et de documents WPS-PLUS 
   (V4.2 et ultérieure) et WordPerfect[R] (V4.2 et ultérieure) au 
   moment où ils sont transmis au client.

o  Disponible sous OpenVMS Alpha et VAX.


PARTAGE DES ARCHIVES

L'option de partage des archives est comprise dans Office Server en 
standard et permet aux utilisateurs de systèmes distants d'accéder 
à des documents placés dans des tiroirs et des dossiers spécifiés, 
comme s'ils étaient disponibles en local.
A mesure que les documents multimédia de tout type deviennent plus 
grands et que les listes de distribution s'allongent, la possibilité 
d'accéder directement aux informations réduit la charge sur le système 
de messagerie et permet une amélioration générale des performances. 
En général, les groupes d'utilisateurs sont avertis de l'existence 
d'un document dans un dossier local par un message, qui offre le 
choix d'y accéder ou pas.

Les utilisateurs ayant les privilèges appropriés, les administrateurs 
et les gestionnaires du système sont en mesure de définir des listes 
et des contrôles d'accès, afin de garantir que seuls les individus et 
les groupes d'utilisateurs autorisés peuvent accéder à des documents 
et messages situés dans tiroirs et des dossiers définis.

Dans ALL-IN-1, cette fonction correspond à l'option Distributed 
Sharing (SPD : 36.97).


UTILITAIRE DE MIGRATION MAILWORKS

Office Server V6.0 comprend des outils de migration serveur qui 
permettent la migration de données utilisateurs et la mémoire de 
messages MailWorks OpenVMS vers Office Server. Les utilisateurs 
peuvent installer Office Server sur site sur leur systèmes VAX et 
Alpha ou sur de nouveaux systèmes Alpha et copier la mémoire de 
messages en format Office Server, ce qui utilise beaucoup d'espace 
disque ou renommer la mémoire de messages sans nécessité d'espace 
disque supplémentaire. La copie du contenu garantit que 
l'environnement MailWorks original reste inchangé.

Remarque : Il est vivement recommandé de faire une copie de 
           sauvegarde complète du système MailWorks avant 
           d'effectuer la migration vers Office Server.

L'utilitaire de migration comprend :

o  Un outil de vérification avant la migration. Cet outil détecte les 
   données importantes pour la migration et tout conflit pouvant 
   survenir.

o  Une capacité de migration simultanée d'un ou de plusieurs 
   utilisateurs, effectuée par l'administrateur système.

o  Une capacité de migration de profils utilisateurs, d'archives et 
   de carnets d'adresses personnels.

o  La préservation de l'intégrité des messages.

Des instructions détaillées relatives à la migration depuis MailWorks 
OpenVMS étaient contenues dans le kit de mise à jour Office Server 
V4.0A et sont également comprises dans le kit complet V6.0.

ARCHIVES/GESTION DES ARCHIVES

Les archives Office Server V6.0 vous permettent d'imbriquer des 
dossiers jusqu'à 10 niveaux et de créer des dossiers vides. Il s'agit 
d'un référentiel de fichiers entièrement multimédia. Des documents 
et des messages de tout type (HTML, MP3, PostScript[R], DOC, XLS, 
PPT, MPP, JPEG, MPEG, vidéo, audio, ASCII, ainsi que d'autres formats) 
peuvent être classés dans les archives.


Tolérance des défaillances et Répartition des charges

Les archives tolèrent les défaillances, ce qui signifie que dans le cas 
d'un redémarrage dû à une panne du serveur des archives ou du serveur 
AIDA, Office Server tente automatiquement de se reconnecter aux clients 
TeamLinks V4.5, V5.0 et Outlook.

Il est également possible d'indique le nombre maximum de connexions 
client à chaque serveur FCS et AIDA, permettant ainsi de répartir 
la charge entre plusieurs serveurs, UC et noeuds d'un cluster. De plus, 
des clients donnés peuvent être automatiquement connectés aux serveurs 
peu chargés, assurant ainsi des temps de réponse très performants à des 
utilisateurs précis.

La gestion de ces fonctionnalités s'effectue à partir du client 
Administrator et n'est pas réalisable depuis un VT.


Fonctionnalités établies

Les archives sont basées sur un système de partage selon lequel, 
lorsqu'un message est envoyé à plusieurs destinataires sur le système 
local, celui-ci n'est pas copié pour chaque destinataire, mais des 
pointeurs permettent aux destinataires de consulter une copie unique du 
message. Ce système de copie unique offre deux avantages, à savoir 
une meilleure performance et une réduction de l'espace disque requis.

Les archives ont la même fonction qu'une armoire de bureau : 
les tiroirs contiennent des documents placés dans des classeurs. 
Chaque classeur peut avoir jusqu'à dix niveaux d'imbrication, 
proposant de la sorte une méthode plus structurée pour conserver les 
informations. Les classeurs sont rangés par ordre alphabétique et 
les documents par date de création, le premier document étant le 
plus récent.


Dans un système utilisant les Classeurs imbriqués, les utilisateurs 
VT pourront encore consulter tous les classeurs, et accéder à tous 
les messages et les documents comme dans les versions précédentes. 
Cependant, les Classeurs imbriqués seront précédés du symbole "~".

Les informations associées à chaque document, telles que le titre, 
la date de création, le numéro du document, l'auteur, les mots-clés 
et le type de structure sont mises à jour dans les archives. 
Des options permettent à l'utilisateur de mettre à jour et de gérer 
le contenu de ses archives. Celui-ci peut imprimer ou détruire 
plusieurs documents à la fois. Celui-ci peut imprimer ou détruire 
plusieurs documents à la fois. Il lui est également possible de 
sélectionner des documents en fonction de critères qu'il aura lui-
même définis.

Les documents peuvent être reclassés, réaffectés à plusieurs endroits 
ou copiés, individuellement ou par classeur. Les informations qui leur 
sont associées, telles le titre et les mots-clés, peuvent être modifiées. 
Lorsqu'un document est réaffecté à plusieurs emplacements au sein d'un 
même classeur, seule une copie du corps du document est stockée afin de 
minimiser l'espace de stockage.

Les documents détruits par l'utilisateur sont placés dans la Corbeille, 
jusqu'à ce que celle-ci soit vidée de son contenu.

Les archives permettent d'accéder à des documents par un mécanisme 
commun, indépendamment du format de ces documents. Tous les types de 
document sont virtuellement pris en charge, y compris les documents 
multimédia. En outre, tous les attributs X.400 P2 peuvent être stockés 
dans les archives Office Server.

La fonction de réservation consent à chaque utilisateur de protéger un 
document auquel il souhaite apporter un certains nombre de changements, 
sur site ou à distance, afin d'empêcher d'autres utilisateurs de le 
modifier. Lorsqu'un document est réservé, les autres utilisateurs 
peuvent lire ce document sans l'éditer.

Le système de traitement de documents opère une distinction nette 
entre le format révisable et le format final des documents. Ce système 
distingue donc le formatage et l'impression, ce qui permet d'effectuer 
le formatage indépendamment de l'impression.


Mail Janitor

Une fonction mail janitor utility est incluse au produit de base. Elle 
permet à l'administrateur système de reclasser les anciens messages 
des classeurs READ et OUTBOX vers le classeur WASTEBASKET de 
l'utilisateur. L'administrateur peut notamment définir une durée de 
conservation maximale des messages dans ces classeurs (60 jours par 
défaut), ainsi qu'un nombre maximal de messages à détruire par classeur.
Des options définies dans le profil et la configuration permettent de 
prévoir (ou d'annuler) les actions par défaut en fonction du nombre 
de messages et de la durée d'absence de l'utilisateur.


La gestion de la fonction Mail Janitor peut être effectuée depuis le 
client Administrator V5.0 ainsi que depuis VT.


Checkmail

Cette fonction permet de vérifier, avant le lancement d'Office Server, 
si des messages non lus ou non envoyés figurent dans le tiroir par 
défaut et d'en informer l'utilisateur, le cas échéant.


Zones communes

Les utilisateurs peuvent créer des tiroirs communs permettant à 
d'autres utilisateurs de lire et d'éditer des documents partagés. 
Des paramètres de gestion de l'accès permettent au propriétaire 
d'un tiroir de déterminer quels utilisateurs ou groupes 
d'utilisateurs ont accès à ce tiroir et les opérations qu'ils 
peuvent effectuer. Certaines actions simultanées sont interdites 
lorsqu'un utilisateur modifie un document commun. Ainsi, les autres 
utilisateurs qui tentent d'éditer le même document sont informés 
que le document est en cours d'utilisation.

La fonction de réservation permet à un utilisateur d'Outlook de 
TeamLinks ou de VT de réserver un document dans les archives avant 
de le télécharger sur sa station de travail, indiquant ainsi à tous 
les autres utilisateurs ayant accès à ce document qu'il est en 
cours de modification. Toute personne essayant par la suite 
d'éditer le document est informée que celui-ci est réservé par une 
autre personne, et que seul un accès en lecture est permis. 
Tout autre type d'accès ne sera possible qu'une fois le document 
modifié et libéré.

Les utilisateurs disposant des paramètres de profil appropriés 
peuvent définir un ensemble d'utilisateurs en tant que groupe. 
Des groupes multiples peuvent être créés. Les groupes disposent de 
privilèges d'accès précis aux documents ou aux tiroirs communs. 
Des membres peuvent être ajoutés ou retirés d'un groupe et les 
droits d'accès associés sont automatiquement attribués ou retirés. 
Les utilisateurs peuvent aussi vérifier à quel groupe ils 
appartiennent.


MESSAGERIE ELECTRONIQUE

Les informations suivantes sont spécifiques à l'accès au client 
Office Server VT et décrivent les fonctions disponibles. La plupart 
de ces fonctions sont accessibles à partir de TeamLinks et 
d'applications utilisant MAPI. Consultez le document SPD 
correspondant pour obtenir une liste complète des fonctionnalités 
de chaque produit.

La messagerie électronique permet aux différents utilisateurs de 
communiquer. Ceux-ci peuvent créer, éditer, lire, imprimer, détruire, 
envoyer et ré-acheminer des messages. Les en-têtes de messages 
(c'est-à-dire le destinataire, l'objet et la classification) peuvent 
être édités de la même façon que le texte du message avant l'envoi.

Lorsqu'il crée un message, l'utilisateur doit en indiquer le(s) 
destinataire(s). Facultativement, il peut également indiquer les 
destinataires auxquels il souhaite faire parvenir une copie, ou une 
copie conforme invisible (CCI), du message. 
En outre, il peut spécifier une date d'expiration, ainsi que 
l'importance (Faible, Normale ou Grande) et la classification (Non 
classifié, Confidentiel, Privé ou Personnel) de chaque message envoyé. 
La messagerie électronique aussi offre la possibilité d'utiliser des 
diminutifs (connus de l'expéditeur uniquement) ainsi que des listes 
de distribution.

L'utilisateur peut répondre aux messages reçus et les ré-acheminer 
(ou les deux). Les réponses peuvent être adressées uniquement à la 
personne ayant envoyé le message ou à tous ses destinataires. Il peut 
également demander confirmation de la réception ou de la lecture du 
message et affecter différentes priorités d'expédition : express, 
normal ou lent. Il peut programmer l'envoi d'un message à une heure 
déterminée.

Les messages sont affichés et traités exactement comme les autres 
documents. Ils sont répertoriés dans des classeurs par Office Server 
en fonction de leur état : arrivé, lu, créé ou envoyé.
Lorsqu'un message a été envoyé, un exemplaire de ce message (par 
système ou par grappe) est partagé par l'expéditeur et le destinataire. 
Cet exemplaire ne peut être édité. Il peut cependant être reclassé 
dans un ou plusieurs classeurs des archives de l'utilisateur.

Un message classé dans les archives peut être envoyé à d'autres 
utilisateurs. Il est possible de faire suivre automatiquement vers 
un autre destinataire tous les messages qui arrivent ou encore, de 
créer un message de réponse automatique qui sera envoyé en réponse 
à tout le courrier entrant (par exemple type "Je suis absent 
jusqu'au. . .").

Toutes les opérations effectuées par les utilisateurs s'exécutent 
correctement dans des environnements communs et décentralisés. Les 
opérations de messagerie se déroulent toujours dans le tiroir 
PRINCIPAL de l'utilisateur, quel que soit le contexte précédent.

Office Server peut afficher l'index des documents joints à un message, 
ainsi qu'un groupe d'options destinées à l'impression ou à la lecture 
de chaque document joint.
Les documents joints peuvent être classés comme texte révisable ou 
comme messages à réexpédier.

L'option Accès à la messagerie d'un utilisateur (AMU) permet à un 
utilisateur d'effectuer des opérations de messagerie au nom d'un 
autre utilisateur, comme une secrétaire pour un directeur. L'option 
Autorisation d'accès aux messages (AAM) permet à un utilisateur 
d'autoriser d'autres utilisateurs à traiter son courrier. 
L'en-tête du message contient les deux noms d'utilisateur lorsqu'un 
message est envoyé.

La fonction d'adressage inclut :

o  La reconnaissance automatique d'une adresse Internet.

o  Un support X.400 intégral.

o  Le support continu de l'adressage de type Routeur de messages pour 
   une compatibilité amont et un minimum de gêne pour l'utilisateur, 
   même si le système de messagerie a été remplacé par MAILbus 400.

o  La possibilité de créer une liste de distribution à partir d'une 
   liste d'adresses contenues dans l'en-tête d'un message reçu.

o  Le support des diminutifs (même longs), la manipulation des adresses 
   électroniques longues sur les messages envoyés ou reçus par 
   l'intermédiaire de passerelles vers d'autres systèmes de messagerie, 
   et la possibilité de créer un diminutif pour l'expéditeur d'un 
   message.


ANNUAIRES

Les annuaires facilitent l'accès et la mise à jour d'informations 
indispensables aux utilisateurs. L'utilisateur a la possibilité de 
consulter un annuaire de numéros de téléphone et d'adresses qui est 
géré par l'administrateur système. Un annuaire de tous les utilisateurs 
d'Office Server peut également être consulté.

Il est également possible de constituer des listes personnelles de 
numéros de téléphone, de diminutifs et des listes de distribution.

Avec l'adoption de MAILbus 400 comme système de messagerie, les 
services d'annuaire sont assurés par Enterprise Directory for eBusiness 
aka Compaq X.500. Veuillez vous référer au SPD 40.77.

Si vous installez Enterprise Directory for eBusiness ou  MAILbus 
Directory Service, Office Server permet d'effectuer des recherches 
dans les annuaires de la messagerie en fonction de différents critères 
tels que le nom, le service, le lieu de travail et la société.

Voir aussi section sur le support LDAP, plus haut dans ce document.

IMPRESSION

La fonction d'impression permet à l'utilisateur de préciser 
la destination du document, le nombre d'exemplaires et le format 
d'impression. Il peut lancer l'impression vers n'importe quelle 
imprimante accessible aux utilisateurs, dans un autre document 
ou au terminal. L'impression étant gérée en arrière-plan par 
le système, le terminal reste disponible pour tout travail non lié 
à l'opération en cours.

Une fonction de vérification du type d'imprimante permet de déterminer 
si l'imprimante demandée accepte le format final du document.

Au cours d'une opération d'impression, il est possible de 
sélectionner les options courantes et les options LPS et PS (LPS20, 
LPS40, PS17 et PS32). Pour une plus grande facilité d'utilisation, 
il est préférable de recourir aux styles d'impression définis par 
le système ou par l'utilisateur.

Suppression de l'impression d'une liste de distribution

La suppression d'une liste de distribution lors de la lecture et/ou 
de l'impression d'un message dépend des préférences définies par 
l'utilisateur.

GESTION DES FILES D'ATTENTE

Office Server permet aux utilisateurs d'intervenir sur les travaux 
d'impression, les travaux différés et ceux de type serveur. Grâce aux 
fonctions de sélection multiple dans les index, les utilisateurs 
peuvent obtenir des informations sur des travaux, les supprimer, 
éditer leurs attributs et redémarrer ceux qui sont en cours 
d'exécution.

La gestion des files d'attente et des travaux peut être effectuée 
par Office Server. Le responsable système peut affecter un ou 
plusieurs utilisateurs à la gestion de files d'attente locales et 
réacheminer les travaux sur des files d'attente différentes.

CONVERSION DE DOCUMENTS

Office Server prend en charge les types de données suivants :

o  Rich Text Format (RTF)

o  WK1, WK3, DIF, MacWrite[R], DX(tm), WPL, PICT, WordPerfect

o  PS - format de document final PostScript

Office Server permet de lire tous ces formats, à l'exception de 
PostScript. La fonction de conversion CDA(tm) est utilisée pour 
convertir de/vers les types de données mentionnés ci-dessus.

Les utilisateurs peuvent transmettre et recevoir ces types de 
documents par l'intermédiaire de la messagerie, les classer dans 
leurs archives, ainsi que les lire, les imprimer, les copier et 
les convertir. La qualité de l'affichage de ces documents dépend 
des utilitaires CDA propres au système OpenVMS. Office Server 
utilise le formateur de caractères CDA, ce qui donne de meilleurs 
résultats et permet l'inclusion des documents qui font l'objet de 
références externes.

Office Server imprime les documents CDA en PostScript si 
l'imprimante de destination accepte ce format. Si ce n'est pas 
le cas, Office Server convertit les documents au format ASCII 
avant l'impression.

Lorsqu'un utilisateur tente d'éditer un document CDA à l'aide 
d'un éditeur standard, il obtient un message l'informant que ce 
type de document n'est pas reconnu. Dans ce cas, il a la 
possibilité de convertir le document CDA dans un format accepté, 
puis de l'éditer. Cette opération peut toutefois faire disparaître 
certains attributs de formatage.

Les conversions sur Alpha nécessitent l'installation de 
DECwindows(tm) V1.2, et la version minimum requise de 
CDA$ACCESS.EXE et DDIF$CC_VIEWSHR.EXE est V1.8.

Les conversions sur VAX nécessitent l'installation de CDA 
Converter Library V2.2 qui installe également CDA Run-Time 
Services V1.0. La version minimum requise de CDA$ACCESS.EXE et 
DDIF$CC_VIEWSHR.EXE est V1.7.

Client Access Licensing

Client Access Licensing (CAL) présente l'avantage financier d'accorder 
à un utilisateur les droits d'accès au système Office Server et aux 
clients Compaq avec une seule licence. Par exemple, une licence CAL 
permet à un utilisateur d'accéder aux fonctionnalités d'Office Server 
à partir de Compaq TeamLinks, de Compaq MAPI Driver, d'un explorateur 
Web ou d'un client VT sans faire l'acquisition du logiciel client 
lui-même.
Cependant, si un client est développé par un fournisseur tiers, les 
droits d'installation de ce client sur le PC doivent être acquis 
auprès de ce fournisseur, étant donné que ceux-ci ne sont pas accordés 
par la licence Compaq Office Server CAL.

Une licence CAL peut sans exception être acquise par chaque utilisateur 
enregistré dans le profil Office Server, c'est-à-dire, qu'à tout 
moment, un utilisateur enregistré doit être disposer d'une licence CAL. 
Les CAL peuvent être déplacées d'un système à un autre selon la 
fluctuation du personnel, mais le nombre de CAL présent sur chaque 
site doit être égal ou supérieur au nombre d'utilisateurs enregistrés 
du site.


Remarque : Dans les clusters OpenVMS, les licences traditionnelles 
           et la licence Client Access Licensing ne peuvent pas 
           coexister. Par exemple, un système Office Server 
           fonctionnant sous licence CAL ne doit pas être ajouté 
           à un cluster Office Server fonctionnant sous les licences 
           à usage illimité ou utilisateur.


Licences système Office Server

Outre les licences CAL, chaque installation Office Server doit 
disposer d'une licence (système) Office Server (UPI 5TB). 
Pour l'utilisation d'Office Server sur des clusters OpenVMS, une 
licence système (QM-5TB9A-*A) est requise pour chaque système OpenVMS 
du cluster.

Chaque kit Office Server (QB-5TBPA-SA) comprend la licence, le support 
et la documentation Office Server, ainsi que le support, la documentation 
et la licence pour installer Office Server Administrator client 
(UPI 5TC). Des clients Administrator supplémentaires peuvent être 
installés avec l'acquisition de licences supplémentaires (QM-5TC9A-AA).
